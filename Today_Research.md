# Day 9: AI活用によるWEBサイト制作② リサーチ結果

## 1. 概要と目標
- **対象コンテンツ**: 「コース情報共有」Day9 (AI活用によるWEBサイト制作②)
- **課題感（ユーザー指摘事項）**:
  - 動画内では「センス不要」「詳細なプロンプトを投げるだけで簡単に作れる」と語られがちだが、**「そもそも非エンジニアが詳細なプロンプトを正確に書けるなら苦労しない」**という現実とのギャップがある。
  - **「トークン消費（利用制限）」に対する説明が圧倒的に不足している**。欲張って色々な機能を詰め込んだり、「なんか違う」と曖昧な指示を繰り返したり、壁打ちから実装・デバッグまですべてをAIに任せきりにすると、無課金でも有料版でも（Antigravityでさえ）あっという間に利用制限に引っかかってしまう。
  - AIを「魔法の杖」として安易に扱うのではなく、上記のリスクや現実的な制約を加味した上で、地に足の着いた教材に落とし込む必要がある。
- **目標**: 自律型AIエージェントの強力さを伝えつつも、プロンプト作成の難しさやトークン消費の罠、そして「静的サイトの制約」といった現実的な落とし穴を丁寧に解説する、実践的かつ啓蒙的なチュートリアルページを構築する。

## 2. コンテンツ構成と実装要件
これまで通りのフォーマットを踏襲し、以下の構成でマークアップを行う。各パートの最下部に「次へ（後半へ）」「ホームへ戻る」等のナビゲーションボタンを配置する。

### 【前半：自律型AIエージェントの活用と現実的な壁】
- **動画埋め込み**: 指定URL（3本）を、プロジェクトルールの「YouTube Facadeパターン」を用いて配置。
- **共有された自律型AIエージェントとツールの紹介**:
  - Manus, Claude Co-Work, Google Stitch, Claude Code などの各ツールの特徴とリンクを見やすくカードUI等で整理する。
  - 制作フロー（NotebookLMで設計 → ManusやStitchで構築・公開）の図解的表現。
- **⚠️ 現場のリアル：AI活用における「落とし穴」の解説（最重要・ユーザー指摘対応）**:
  アラートやコラム枠を使い、以下の現実的な課題を解説する。
  1. **「詳細なプロンプト」の難しさ**: 非エンジニアが最初から完璧な指示を出すのは至難の業。「センス不要」の裏には、要件を論理的に言語化する「国語力」や「設計力」が求められること。
  2. **トークン消費の罠（利用制限）**: あいまいな指示で修正を繰り返したり、壁打ちから実装、デバッグまで全てを一気にやろうとすると、あっという間にAIの「利用制限」に達してしまうこと。目的を絞って段階的に活用する重要性。
- **実習：制約条件付きでWEBサイトを作ってみよう**:
  - 動的サイト（サーバー処理）と静的サイト（HTML/CSS/JSのみ）の違い。
  - GitHub Pages等の静的ホスティングではサーバー処理が動かないため、「静的サイト前提」「サーバー処理なし」という**ネガティブプロンプト（制約条件）**を明示することの重要性。

### 【後半：エラーへの対処と公開プロセス】
