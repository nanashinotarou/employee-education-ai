# Day 9 後半: Claude Code 超入門（ガチ勢向け深掘りリサーチ）

## 1. 動画の概要と目的
* **対象動画**: 【Claude Code超入門】実は簡単？！Claude Codeで実際にWebサイトを作成する様子を初心者でもわかるように解説させていただきます
* **意図**: 本コースの一般受講生は無料ツールで済ませる可能性が高いが、「ガッツリ使い込みたい」ユーザー向けに、動画の内容をかみ砕いて網羅的にテキストで解説するセクションを設ける。

## 2. 網羅すべき解説内容

### ① Claude Code とは？（Cursorとの決定的な違い）
* **Cursor**: エディタ（IDE）。人間がコードを開きながらAIに補完・修正を指示する「AIペアプログラマー」。
* **Claude Code**: ターミナルで動く「自律型AIエージェント」。エディタに依存せず、ディレクトリ全体を把握し、自らファイルを作成・修正し、テストを実行し、エラーを読んで自律的に修正する。

### ② 導入手順
1. Node.js のインストール（前提条件）。
2. ターミナルで `npm install -g @anthropic-ai/claude-code` を実行。
3. `claude auth` コマンドで認証（AnthropicコンソールでのBilling設定が必須）。
4. ターミナルで作業ディレクトリに移動し、`claude` と打って起動。

### ③ 必須・便利な内部コマンド（スラッシュコマンド）
* `/help` : コマンド一覧の確認。
* `/compact` : **【超重要】** 長くなった対話履歴を圧縮し、トークン消費を節約する。
* `/clear` : 現在のセッションの会話履歴を完全に消去し、真っ新な状態に戻す。
* `/cost` : 現在のセッションでのAPI利用料金（推定）を確認する。

### ④ トークン（課金）を抑えるためのベストプラクティス
* **小さなディレクトリで起動する**: プロジェクトのルートではなく、修正したい機能があるサブディレクトリ内で起動することで、無駄なファイルの読み込み（トークン消費）を防ぐ。
* **不要なファイルは `.claudeignore` で除外**: 画像やビルド成果物などAIに読ませる必要のないファイルは除外する。
* **こまめな圧縮と消去**: 対話が一区切りつくたびに `/compact` または `/clear` を打つ。
* **設計は「定額制」で済ませる**: いきなりClaude Codeに設計からやらせるのではなく、Web版ClaudeやGeminiで「設計書」を作り、Claude Codeには「この指示書通りに実装して」と渡す（※本コースが推奨する「AIディレクター」の立ち回り）。
