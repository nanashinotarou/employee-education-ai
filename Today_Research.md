# Day 12 リサーチ結果

## 1. 今回のテーマと「ユーザーの悩み」に対する回答
Day11の「システム設計・バックエンド理解」から一歩進み、Day12では**「実際にユーザーが触れる画面（フロントエンド）をAIに実装させ、一つのアプリケーションとして完成させる」**プロセスを体験します。

受講者が抱える「AIがコードを出してくれたけど、それをどうやって実際の画面に反映させるのか分からない」という悩みを解消するため、Claude（Claude Code や Cursor等）を活用したUI構築と、Googleの最新ツール「OPAL」を用いた直感的な実装アプローチを解説します。

**【Day12の核心ポイント】**
- **Claudeによるコンポーネント実装:** 自然言語で「どんなUIにしたいか」を指示し、フロントエンドのコード（HTML/CSS/JSやReact等）を生成・反映させる手法。
- **実装と動作確認の反復:** AIに出力させたコードをプレビューし、エラーやデザインの崩れがあれば、そのままAIにフィードバックして修正させる「プロンプトエンジニアリングの最終形態（デバッグループ）」を学ぶ。

## 3. 実装計画（ClaudeCode）に向けた指示・デザイン要件
以下の要件を踏まえて、ClaudeCodeは `Today_Plan.md` を更新し、Cursor等での実装計画を立ててください。

- **トーン＆マナー:**
  研修サイト全体（`vol01-1.html` 等）のデザインシステムを継承。Day12としての「アプリ開発の集大成」感を出すため、読みやすく洗練されたレイアウトを意識する。
- **必須のページ構成:**
  これまでのDayと同様に、以下の形式を厳守すること。
  1. **本日の目標**
  2. **前半：**（動画Facade + 実習内容）
  3. **後半：**（動画Facade + 実習内容 + OPALリンク）
  4. **今日のまとめ**
- **重要なUIコンポーネント:**
  - **YouTube Facadeパターン:** `GEMINI.md` に記載の通り、動画埋め込みは必ずFacadeパターン（`.vc-thumb` 等）を用い、`maxresdefault.jpg` の背景設定と遅延読み込みJSを実装すること。
  - **ナビゲーションボタン:** 各パート（前半・後半）の最下部には、必ず「次パート」や「ホーム（`index.html`）」へ戻るボタンを配置すること。
  - **モバイル最適化:** `.container` の `padding` 削減など、スマートフォン表示用のメディアクエリ（`@media (max-width: 768px)`）を `<style>` 末尾に必ず含めること。
- **手戻り防止（トークン節約）:**
  コンテキストを圧迫しないよう、既存の `vol11-1.html` や `index.html` の構造・CSSクラスを再利用し、一発で動作する `vol12-1.html` を作成する計画を立てること。
