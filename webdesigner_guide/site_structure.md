## このサイトについて

### コンセプト
「デザインセンスはある。でもWebはまだわからない」という人が、  
Webデザイナーとして活動するために必要な知識を体系的に学べる学習サイト。

コーダーやエンジニアを目指すのではなく、**デザイン・営業・ディレクション**の立場で  
Web制作に関わる人に向けた内容に特化する。

#### サイトを貫くメッセージ（設計の根幹）
**「素材屋は儲からない。プロジェクト屋になれ。」**

イラスト・画像・動画・音楽──どれだけ素材を生成・制作できても、  
「言われた作業をこなすだけ」では単価は上がらない。  
技術力よりも「仕事の全体を動かす力」を持った人材が、今もっとも必要とされている。

このサイトは単なるWebデザイン入門ではなく、  
**受講者がすでに持つデザイン力に「プロジェクト遂行力」を掛け算するための教材**。  
CanvaやFigmaが使えるなら、そこに上流設計・クライアント対応・AI活用が加わったとき、  
仕事の幅と採算は根本から変わる。

### ターゲット
- グラフィックデザイン経験があり、Webデザインの仕事に本格参入したい方
- Canva / Figmaは触っているが、Web制作の全体像が掴めていない方
- コーダーではなく、クライアント対応・ディレクション役として活躍したい方
- 社員研修（2026年4月）の受講者

### 既存研修サイトとの関係
- 既存の `index.html`（研修カレンダー）にリンクボタンを追加し、このサイトへ誘導
- 本サイトは `webdesigner_guide/` 配下にすべてのファイルを置く
- 独立したサイトとして成立するが、研修サイトからアクセスできる構成

### メイキング特典：ワイヤーフレーム保存
- 本番サイトとは別に、制作途中のワイヤーフレーム状態を `webdesigner_guide/wireframe/` 配下に保存
- 本番サイトから「制作の裏側を見る」リンクで誘導
- 教材として「設計書 → ワイヤー → 本番」の全プロセスが辿れるようにする狙い

## 各ページの詳細構成

### トップページ（index.html）

**目的**：サイト全体のハブ。メッセージ・ターゲット・「読んだら何になれるか」を先に明示してから、コンテンツへ誘導する。

**UX方針**：ヒーローにCTAボタンは置かない。コンテンツを最後まで読んだ人に改めて二次CTAで誘導。

```
[ ヒーローエリア ]

  サブ：「素材を作れるだけでは、仕事にならない時代がきた。
         Webの知識を掛け算して、プロジェクトを動かせる人材へ。」
  CTAなし → スクロール誘導（↓ まずはこのサイトについて）

[ このサイトについて ]  ← ★ 新設セクション（3ブロック構成）
  A. こんな人のために
     - グラフィック経験はあるが、Webに越境したい方
     - Canva / Figmaは触れるが、Web制作の全体像が掴めていない方
     - 自分でコードは書かないが、クライアント対応・ディレクションでWebに関わる方
     - 「言われた作業をこなすだけ」を卒業したい方

  B. なぜ作ったか
     - 素材を作るだけでは採算が取れない（動画編集1本5000円の現実）
     - 定型の作業をどれだけ積み上げても、素材納品だけでは単価の天井がある
     - 逆に、コーディング力が初歩でも「プロジェクトを動かす力」があれば仕事は変わる
     - AIが実装を担う今こそ、「何を依頼するか・成果物が妥当か」を判断できる人が価値を持つ
     - このサイトは「Webの技術を教える場」ではなく、「仕事の解像度を上げる場」

  C. 読み終えたら、あなたは
     - クライアントと対等に要件定義ができる
     - 制作者やAIに的確な指示が出せる
     - 成果物のクオリティを自分で判断できる
     - 案件の採算（単価・作業時間・コスト配分）を見極められる
     - 「素材屋」ではなく「プロジェクトを完遂するディレクター」として仕事を取れる

### Page 06：クライアントワークの実際（page06_client.html）

```
[ ページタイトル ]
[ クライアントとの出会い方（SNS・紹介・クラウドソーシング）]
[ はじめての打ち合わせで聞くこと（ヒアリングシート雛形）]
[ 提案・見積もりの考え方 ]
[ 制作中のコミュニケーション（報連相・確認の取り方）]
[ トラブルを防ぐための契約・仕様書の話 ]

[ ★ COLUMN：作画カロリーと採算性の話 ]  ← 新設
  ─────────────────────────────────
  「作画カロリー」= その案件にかかる設計・実装・調整コストの総量。
  高カロリー案件を低単価で受けることが、最大のリスク。

  ▼ 作画カロリー高い案件（単価が見合わないと危険）
  - 複雑なアニメーション・インタラクション多用のサイト
  - カスタムイラスト・オリジナルビジュアルが必須
  - ページ数が多く、コンテンツ量も大きいコーポレートサイト
  - 多言語対応、会員機能など複合要件
  - 「とりあえず作って」「なんとなくいい感じで」系のクライアント

  ▼ 作画カロリー低く・高単価を狙える案件
  - LP（1ページ完結・明確な目的あり）
  - WordPressテーマのカスタマイズ（ゼロ実装不要）
  - STUDIOやwixなどノーコードCMS案件
  - 定期保守・更新作業（継続課金モデル）
  - 要件定義・ワイヤーフレームのみ（上流工程特化）

  → カギは「省エネで回せる案件設計力」。
    AI活用を前提にすると、コーディングコストは下がるが
    設計・ディレクション・確認コストは変わらない。
    だから上流工程の価値はむしろ上がっている。

[ ★ COLUMN：AI開発環境のコスト配分と採算設計 ]  ← 新設
  ─────────────────────────────────
  開発にかかるコストは「時間」だけではなく「AIのトークン使用量」も含む時代。
  
  ▼ 現実的なAI開発環境の例（月額目安）
  - Claude Code（Pro）：約 ¥2,500〜3,000
    → 要件定義・実装計画・レビュー・仕様書作成に集中使用
  - Cursor（Pro）：約 ¥2,000〜3,000
    → 実装（コーディング）担当
  - Antigravity等（Googleエコシステム連携）：約 ¥数百〜数千
    → デプロイ・Google Workspace連携・自動化
  合計：月 ¥10,000 前後で疑似的な「止まらない開発環境」が成立

  ▼ 考え方
  - MAXプランや高コストAIを無制限に使っても採算割れなら意味がない
  - 「作業を分担・分散させる設計力」が収益を決める
  - Claude Code → 計画と判断、Cursor → 実装、Antigravity → 運用
    という役割分担が"省エネ高単価"の一形態
  - AIのトークン消費量は、案件の複雑さ・開発難度にある程度比例する
    つまり：「AIに何を頼むか」を設計できる人こそが、コストを制御できる

  ▼ 採算を見る目安（参考）
  | 案件種別               | 目安単価         | 目安工数   | 難易度 |
  |------------------------|-----------------|-----------|--------|
  | LP（シンプル）         | 3〜8万円        | 3〜10h    | ★☆☆  |
  | コーポレートサイト     | 15〜50万円      | 40〜100h  | ★★☆  |
  | WordPress構築          | 10〜30万円      | 20〜60h   | ★★☆  |
  | ECサイト               | 30〜100万円+    | 100h+     | ★★★  |
  | 保守・更新（月額）     | 1〜5万円/月     | 数h/月    | ★☆☆  |
  | 上流工程のみ（要件定義・WF）| 5〜20万円 | 10〜30h   | ★★☆  |

[ 次のページへ→ ]

## 研修サイト（index.html）への追加

既存の `index.html` のカレンダーエリア下部または右側パネルに  
以下のリンクボタンを追加する：

```html
<a href="webdesigner_guide/index.html" class="guide-link-btn">
  📘 How to Webデザイナー ガイドを読む
</a>
```

スタイルは既存サイトのデザインに合わせてカスタマイズ。
